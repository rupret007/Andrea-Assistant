# Ghostwriter

Like `/voice-to-content` but for long-form. Full essays, not captions.

## When to use

- "Draft a blog post on {topic}"
- "Write a Substack about {theme}"
- "Ghostwrite this — {topic}"
- "Long form on {X}"
- Any 500–2,000 word writing task where Jeff's voice matters

## Process

### Step 1 — Extract the input
Get from Jeff:
- **Topic / angle** — what's it about?
- **Audience** — fans / LinkedIn network / technical peers / general / private journal?
- **Length target** — short (500), standard (1,000), long (2,000)?
- **Key points** — what MUST be in there? (1–5 bullets, rough)
- **Stories / examples Jeff wants to use** — anything specific?
- **Tone calibration** — warm / skeptical / punchy / reflective?

### Step 2 — Load voice calibration
Read:
- `memory/andrea-identity.md` (for Andrea's own voice rules, which echo Jeff's)
- `memory/music/song-catalog.md` — his song titles reveal his humor and style ("Another Bland Love Song", "Trailer Swift", "Tooted Last Tuesday")
- Any existing Drive docs by Jeff in a similar format — search for blog posts, Substacks, LinkedIn posts he's written before

### Step 3 — Outline first
Propose an outline (5–8 bullets) before writing. Don't write the full draft until Jeff approves the structure.

```
📝 **Proposed outline for "{title}"**

1. {hook} — {2-line setup}
2. {section 2} — {what it covers}
3. {section 3} — {what it covers}
4. {turn} — {pivot point, story or insight}
5. {payoff} — {the thing Jeff actually wants the reader to take away}
6. {CTA or close}

Approve / rearrange / skip sections?
```

### Step 4 — Draft
On approval, write the draft. Rules:

- **Open with the hook.** No "I've been thinking about..." throat-clearing.
- **Concrete over abstract.** Real names, real numbers, real memories.
- **Short paragraphs.** 2–4 sentences each. Whitespace is a feature.
- **Jeff's rhythm.** Mix punchy short sentences with occasional longer ones. Vary.
- **No corporate filler.** No "at the end of the day", "low-hanging fruit", "circle back".
- **First-person where it fits.** He's a real person writing, not a thought leader.
- **Plain English defaults.** Avoid jargon unless the audience shares it.
- **End with something that lands.** Not with a whimper.

### Step 5 — Offer variants
After the draft, offer:
- Tighter 60% version
- A different opening hook
- A more punchy or more reflective register

### Step 6 — Deliver
Return the draft in a code-fenced block so Jeff can copy cleanly. Flag anything you invented or extrapolated that he should fact-check.

Save a copy to `memory/library/drafts/{kebab-title}-{YYYYMMDD}.md` for future reference.

## Format templates

### Blog post / Substack
Hook → setup → 2-3 body sections → turn → payoff → close

### LinkedIn long-form
Hook → credential beat (one line why Jeff's voice matters on this) → body → contrarian take → CTA

### Technical write-up
Problem statement → what I tried → what worked / didn't → concrete takeaway → code/artifact link

### Band liner notes / album essay
Origin story → recording memory → song-by-song → looking ahead

### Personal reflection / journal
Moment → zoom in → generalization → small truth

## Never do

- Never publish or send. Draft-only, always.
- Never claim authorship for Jeff in any metadata.
- Never reveal private memory content (life threads, decision log) in public drafts unless Jeff explicitly said to.
- Never fabricate quotes or statistics.
- Never sneak in content Jeff didn't ask for.
