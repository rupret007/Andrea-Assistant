# Library — Jeff's curated knowledge

Andrea's knowledge library. A place to save content she's gathered or Jeff has explicitly asked her to keep — research briefs, reference documents, creative sparks, drafts in progress.

Distinct from:
- **Memory files** (who/what/when — facts about Jeff's world)
- **Tasks** (things to do)
- **Life threads** (ongoing situations)
- **Playbooks** (procedural how-to)
- **Archive** (stale / retired content)

## Structure

```
library/
├── drafts/       # /ghostwriter output stashed for later
├── research/     # /research-topic outputs (duplicated here for archival)
├── creative-sparks.md   # running log of ideas worth revisiting
├── reference/    # curated reference docs Jeff wants to keep
└── README.md     # this file
```

## Rules

- Only save to the library when Jeff says so or when a skill explicitly writes here.
- Never silently overwrite library content.
- Library is Jeff's trust territory — she doesn't prune without permission.

## Related skills
- `/ghostwriter` → writes drafts to `library/drafts/`
- `/research-topic` → saves briefs to `library/research/`
- `/idea-capture` → can route to `library/creative-sparks.md`
- `/knowledge-search` → reads the whole library + memory
